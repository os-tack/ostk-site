REF_INDEX_024 // KERNEL_CLI

COMMAND_INDEX // MAN_PAGES

The ostk CLI is the primary interface for kernel-level agent coordination. It operates on the principle of filesystem-as-truth. Every command is an atomic mutation or a filtered view of the system state. No daemons are required; the CLI is the operator's direct link to the trust chain.

VERSION: v1.0.0-stable
COMPLIANCE: POSIX_ARCHIVE_A
info Direct file mutations are logged to /var/ostk/audit

ostk spawn

[AGENT_INITIALIZATION]
SYNOPSIS
ostk spawn [FLAGS] [REPO_PATH]
Initialize one or more AI agents within a scoped repository boundary.

list PARAMETERS_LIST

FLAG VALUE DESCRIPTION
--count, -c INT Total agents to manifest (default: 1).
--trust-tier 1-4 Security clearance level. Maps to /etc/ostk/tiers.
--isolate BOOL Prohibit inter-agent socket communication.
--trace PATH Export execution telemetry to specific file.

warning CRITICAL // DATA_PERSISTENCE

Spawning agents with --trust-tier 4 allows direct syscall access. The kernel will not intervene in destructive filesystem operations.

→ verify .ostk/rules before execution
